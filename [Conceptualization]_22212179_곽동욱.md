# 1. Business Purpose

## 1.1 Project Background

### 우리, 일반인들은 운동을 얼마나 할까?
![운동을 얼마나할까 사진](https://github.com/KDWorld81/Ex-Log_Docs/blob/main/1-1.png?raw=true)

### 🔴 대한민국 청소년, 운동을 잃어버린 세대
- 질병관리본부의 통계는 우리 사회에 매우 뼈아픈 현실을 보여준다. 하루 60분, 주 5회 이상 활동하는 청소년이 단 5.6%에 불과하다는 수치는, 나머지 94.4%의 청소년이 신체적 활기를 잃은 채 정적인 일상에 갇혀 있는것을 의미한다. 세계보건기구(WHO)가 권장하는 신체 활동 기준과 비교했을 때, 대한민국 청소년의 건강 지표는 사실상 최하위 수준이다.

### 🔴 학업 중심 사회 구조와 학업 이외의 활동의 죄책감
- 이러한 현상의 근본 원인은 대한민국의 고강도 입시 경쟁 구조에 있다. 청소년들에게 운동 1시간은 단순히 건강을 위한 시간이 아니라, 영어 단어 50개를 더 외울 수 있는 시간을 포기해야 하는 기회비용으로 인식된다. 당연하게도 대한민국에서의 공부는 정답이라고 생각들한다.
-  학교와 학원, 독서실로 이어지는 쳇바퀴 같은 일상 속에서 운동은 권장되는 활동이 아닌 사치이거나 노는 여가활동으로 여겨진다. 이 때문에 청소년들은 운동을 하고 싶고, 건강해지고싶어도 심리적 죄책감과 시간적 압박 때문에 자신을 책상앞에만 묶어 두는 현상에 빠진다.
-  이것은 역시나 악순환이기 때문에 계속해서 건강함은 줄어든다.

###  🔴 신체 활동 부재가 가져온 보이지 않는 사회적 비용
- 운동 부족은 단순한 체력 저하에 그치지 않는다. 청소년 비만율의 급격한 상승, 거북목 및 척추 질환의 조기 발생은 물론이고, 에너지를 발산하지 못해 발생하는 극심한 학업 스트레스, 우울증, 불안 장애 등 정신 건강의 위기로 직결된다. 이는 장래 국가적 차원의 의료비 증대와 사회적 활력 저하라는 거대한 청구서로 돌아오게 될 것이 자명하다.

### 🔴 직장인들의 운동 병행의 고충
![직장인들 운동병행 사진](https://github.com/KDWorld81/Ex-Log_Docs/blob/main/1-2.png?raw=true)

- 위의 사진에서 저렇게 가볍게 웹 서핑만하더라도 직장인들의 운동 병행에 대한 고민 글이 넘쳐난다. 일반적인 직장인의 삶을 생각해보면, 아침에 출근해서 퇴근하면 저녁이다. 만약 저녁을 먹지 않고 퇴근한다면? 집에가서 저녁을 먹으면 8-9시가 되버린다.
- 그렇다고 하여, 정시에 퇴근하는 사람이 많을까? 아니면 불규칙한 사람들이 많을까? 불규칙한 사람이 훨씬 많다. 그런 부분에서 직장인들은 전문적으로 운동 하는 사람들처럼 하기가 힘들것이다.
- 청소년들과 마찬가지고, 이들에게도 의무감을 주기보다는 **재미**를 토대로 운동을 할 수 있도록 도와줘야하지 않을까?

### 🔴 기존 앱들의 한계 ? 
- 시중에는 이미 수많은 운동 관련 앱이 존재한다. 하지만 대부분은 전문적인 운동인을 대상으로 서비스를 운영하여 일반인들 입장에서는 어렵거나, 굳이 복잡하게 까지 해서 하고싶지 않아하는 경우도 많다.
- 그리고 단순히 수치를 기록하는 기능에 치중되어 있다. 공부에 지친 청소년들 혹은 일반적인 직장을 다니며 틈틈히 운동을 해야하는 사람들에게는 "건강을 위해 이 동작을 따라 하세요~!"라는 방식은 또 하나의 체육 숙제를 내주는 것과 다름없다.
- 운동을 전문적으로 하지않는 이들에게 필요한 것은 운동을 해야만 할것 같은 의무감이 절대아니다. 의무감은 직장에서의 일이나, 학교에서의 학업 등등 으로 충분한 스트레스를 받을것이다 ..
- 그렇다면 이들에게 필요한것은 무엇일까?
  - 다양한 연령층을 고려한 쉬운 사용법과, 동기부여이다.
  - **예를들어서 게임에 흔히 있는 랭크 시스템을 예로 들면, 해당 랭크를 올리려고 사람들은 열심히 노력하여 게임을 한다. 이것을 나는 운동을하면 경험치를 주고 랭크를 올리는 서비스로 일반인들에게 동기부여를 주는것이다!**
  - 그리고, 사람들의 눈에 확 들어오는 github의 잔디 시스템처럼 운동 한 날마다 잔디가 심기도록 하는 시스템도 고려해보면 좋을것 같다!

### 🔴 마지막으로, 경험 바탕의 제작동기
- 나는 주6일 운동을 매일 갈 정도로 운동을 좋아하는 사람이다. 헬스하면서 항상 느끼는것은 기록은 굉장히 중요하다. 물론 이것은 운동에만 해당되는것은 아닐것이다.
- 하지만 운동할때의 기록은 점진적 과부하를 주는것에는 필수적이다. 그래서 나는 헬스 할때 기록을 많이 하는 편인데, 시중에 있는것들은 그냥 기록 그 자체에 중점을 둔 서비스들이 대부분이다.
-  하지만 그런 서비스는 운동을 좋아하는 사람들을 위한 서비스 라는 생각이 들었다. 운동을 좋아하지 않는다면 기록을 해야하는 이유보다는 기록을 하기 귀찮다는 생각이 우선일것이다. 나의 주변 지인들도 웬만하면 기록은 하지 않는다.
> 💡 이때 든 생각이 일반인들을 위한 동기부여 기록 서비스를 만들어야겠다. 라는 생각을 했다. 
개발을 할수있는것의 가장 큰 장점이 이것이라고 생각한다. 내가 생각한것 원하는것을 실제로 만들수가 있다.
세상에 내가 원하는것을 내놓는다는건 정말 큰 장점이다. 말이 좀 길어졌는데, 해당 서비스의 Background는 이러하다.

## 1.2 Target Market
### 1. 운동을 하기 귀찮은 학생 / 운동을 할 시간이 없다고 생각되는 학생
- 학업 중심의 생활 패턴을 가진 대한민국 중·고등학생, 하루 10시간 이상을 의자에 앉아 보내며 운동을 시간 낭비 혹은 피곤한 일, 쓸모 없는일로 인식하는 94.4%의 학생들
- 체력 저하를 체감하는 수험생, 입시 공부를 위한 기초체력을 다져야겠다고는 하지만, 기존의 고강도 운동 앱이나 헬스장 커뮤니티에 진입 장벽을 느끼는 운동 초보자

### 2. 운동이 귀찮은 운동 할 시간이 없는 직장인 / 번아웃이 온 직장인
- RPG 및 랭킹 시스템에 익숙한 세대, 캐릭터의 레벨업, 티어 상승, 업적(칭호) 달성에서 오는 도파민과 성취감을 즐기는 사람들
- 운동을 단순한 노동이 아닌, **나 자신이라는 캐릭터를 육성하는 게임**으로 생각할수있는 사람들
- 기존 운동앱이 어렵거나 귀찮아서 단순화 된것을 사용하고싶은 직장인

## 1.3 Goals

- 운동 인식의 전환: 운동을 고통스러운 노력, 몸을 만들기 위한 고통에서 즐거운 보상, 성취감을 얻는 행위로 변화시켜, 청소년 신체 활동 및 직장인, 즉 일반인들의 운동 활동률 극대화를 목표로 한다.

- 사용자 리텐션(Retention) 극대화: 휘발성 높은 일반 운동 앱과 달리, 티어 시스템과 칭호 부여 로직, 그리고 잔디 심기 시스템을 통해 사용자가 매일매일 앱에 접속하여 운동을 기록하게 만드는 강력한 중독적 선순환 구조를 구축

# 4. Concept of Operation

## 1) 회원가입

| **항목** | **내용** |
| --- | --- |
| **Purpose** | 서비스를 이용하고자 하는 사용자를 고유하게 식별하고, 개인별로 축적되는 운동 데이터와 성장 지표(경험치, 티어)를 안전하게 관리할 수 있는 개인 공간을 생성함. 특히 실제 사용 가능한 이메일인지 확인하여 보안성을 높이고 허위 계정 생성을 방지함. |
| **Approach** | 사용자는 본인의 이메일 주소와 사용할 비밀번호, 닉네임 등을 입력함. 보안 강화를 위해 이메일로 발송된 임시 인증 번호를 제한된 시간 내에 입력하는 절차를 거치며, 모든 정보가 올바르게 입력되었을 때만 최종적으로 회원 등록이 완료됨. 이 과정에서 사용자의 소중한 개인정보는 안전하게 보호된 상태로 저장됨. |
| **Dynamics** | 처음 실행한 사용자가 서비스를 시작하기 위해 계정 정보를 입력하고, 본인 인증을 위한 이메일 발송 버튼을 누르는 시점에서 발생함. |
| **Goals** | 신뢰할 수 있는 사용자 기반을 확보하고, 각 사용자에게 최적화된 운동 관리 서비스를 제공할 수 있는 시작점을 마련함. |


## 2) 로그인

| **항목** | **내용** |
| --- | --- |
| **Purpose** | 이미 가입된 사용자인지 확인하여 본인의 운동 기록과 획득한 칭호 등에 접근할 수 있도록 권한을 부여함. 로그인 과정을 통해 개인의 프라이버시를 보호하고 본인만이 데이터를 수정하거나 조회할 수 있도록 함. |
| **Approach** | 사용자가 가입 시 등록한 이메일과 비밀번호를 입력하면, 시스템은 저장된 데이터와 대조하여 일치 여부를 판단함. 만약 등록되지 않은 정보이거나 비밀번호가 틀린 경우에는 사용자에게 명확한 안내 메시지를 전달하여 다시 시도할 수 있도록 돕고, 인증에 성공하면 메인 화면으로 진입하여 모든 기능을 사용할 수 있게 함. |
| **Dynamics** | 앱을 재실행했을 때 세션이 만료되어 있거나, 로그아웃 상태에서 자신의 운동 데이터에 접근하려고 할 때 발생함. |
| **Goals** | 사용자의 개인 데이터를 안전하게 보호하고, 끊김 없는 서비스 이용 경험을 제공하는 보안 체계를 확립함. |



## 3) 운동 부위 및 운동 종목 선택

| **항목** | **내용** |
| --- | --- |
| **Purpose** | 사용자가 오늘 수행할 운동을 체계적으로 분류하여 기록할 수 있도록 가이드를 제공함. 부위별로 운동을 나누어 선택하게 함으로써 사용자가 자신의 운동 편향성을 파악하고 균형 잡힌 신체 발달을 도모하게 한다.|
| **Approach** | 사용자는 가슴, 등, 하체 등 6가지 주요 신체 부위 중 하나를 먼저 선택함. 부위를 선택하면 해당 카테고리에 속하는 세부 운동 종목(예: 하체 선택 시 스쿼트, 런지 등) 리스트가 나타나며, 사용자는 그중 자신이 수행한 종목을 직관적으로 선택할 수 있음. 이러한 단계별 선택 방식은 방대한 데이터 중 원하는 항목을 빠르게 찾을 수 있도록 도움. |
| **Dynamics** | 사용자가 운동을 마치거나 시작하기 전, 기록을 남기기 위해 '운동 추가' 메뉴에 진입하여 부위별 아이콘을 터치하는 단계에서 발생함. |
| **Goals** | 사용자에게 명확한 운동 분류 기준을 제시하여 기록의 편의성을 높이고, 체계적인 운동 습관 형성을 지원함. |



## 4) 운동 기록

| **항목** | **내용** |
| --- | --- |
| **Purpose** | 선택한 운동의 구체적인 수행량을 숫자로 입력하여 정량화된 데이터를 축적함. 이 데이터는 사용자가 땀 흘린 노력의 증거가 되며, 경험치로 환산되어 티어를 올리는 가장 핵심적인 근거가 됨. |
| **Approach** | 사용자가 선택한 운동의 특성에 맞는 입력창이 나타남. 근력 운동의 경우 '수행 횟수'와 '세트 수'를 입력하고, 달리기 같은 유산소 운동은 '이동 거리'와 '수행 시간'을 입력함. 시스템은 0이나 음수 같은 잘못된 값이 입력되지 않도록 실시간으로 체크하며, 저장이 완료되면 사용자의 '마지막 운동 날짜'를 오늘로 자동 갱신하여 활동을 기록함. |
| **Dynamics** | 운동 종목 선택을 마친 사용자가 실제 수행한 수치를 입력창에 기입하고 '기록 완료' 버튼을 누르는 시점에서 발생함. |
| **Goals** | 사용자의 노력을 수치화하여 객관적인 성장을 체감하게 하고, 공정한 경험치 부여를 위한 기초 데이터를 확보함. |

| **Goals** | 다양한 수집 요소를 통해 사용자에게 운동의 재미를 더해주고, 특정 부위에 치우치지 않는 다양한 운동 경험을 유도함. |



# 5. Problem Statement

Overview
Ex-Log는 대한민국 청소년의 심각한 신체 활동 부족 문제를 해결하기 위해, 운동 기록을 게임의 '경험치'와 '티어' 개념으로 치환하여 제공하는 서비스이다. 단순히 기록을 저장하는 것을 넘어, 사용자가 책상을 떠나 실제로 몸을 움직이게 만드는 강력한 동기부여를 목표로 한다. 이러한 목표를 달성하기 위해 시스템 설계 및 운영 측면에서 반드시 고려해야 할 주요 문제점들을 다음과 같이 정의한다.

### Problem#1 Motivation & Consistency (동기부여와 지속성)
서비스의 가장 큰 숙제는 운동을 지루한 숙제로 느끼는 94.4%의 청소년및 직장인들을 (통틀어 일반인) 어떻게 지속적으로 참여시킬 것인가이다. 
초기 호기심으로 가입한 사용자가 단발성 이용에 그치지 않도록, 운동의 결과가 즉각적인 보상(경험치 상승, 칭호 획득)으로 이어지는 메커니즘이 정교하게 작동해야 한다. 
만약 보상 체계가 좋지 않다면 사용자들은 해당 서비스를 이용하지 않을것 같다.
따라서 사용자의 활동 수준에 따른 세밀한 레벨 디자인과 수집 욕구를 자극하는 도감 시스템의 구성이 필수적이다.

### Problem#2 Data Integrity & Validation (데이터의 신뢰성과 검증)
Ex-Log는 랭킹 시스템을 통해 동기부여를 유도한다. 
이 과정에서 가장 큰 문제는 입력 데이터의 정직성이다. 사용자가 실제로 운동을 하지 않았음에도 허위로 높은 수치를 입력하여 부당하게 랭킹이나 경험치를 올릴 경우, 정직하게 운동하는 사용자들의 의욕을 꺾고 서비스 전체의 신뢰도를 무너뜨릴수 있을것 같다.
이를 방지하기 위해 상식적인 범위를 벗어난 입력값을 필터링하는 로직이 고려되어야 한다.

### Problem#3 Real-time Feedback & Performance (실시간 피드백과 성능)
사용자는 운동 기록을 저장한 직후, 자신의 경험치가 얼마나 올랐고 티어가 변경되었는지 즉시 확인하기를 원한다.
만약 기록 저장 후 마이페이지에서 반영된 결과를 확인하기까지 긴 대기 시간이 발생하거나 데이터가 갱신되지 않는다면 서비스의 몰입도가 크게 떨어진다. 
빈번하게 발생하는 기록 저장과 조회 요청 속에서도 사용자의 개인 데이터(마지막 운동 날짜, 누적 경험치 등)가 충돌 없이 실시간으로 정확하게 업데이트될 수 있는 효율적 및 성능상으로도 고려한 처리시스템을 갖춰야할것 같다.

### Problem#4 Platform Accessibility (플랫폼 접근성 및 환경 제어)
현재 Ex-Log는 웹 타겟으로 개발되고 있으나, 대한민국 스마트폰 사용 환경은 안드로이드와 iOS 등으로 되어있다. 웹 서비스는 접근성을 저하시킬 수 있다. 
또한, 공부에 집중해야 하는 독서실이나 학교 환경 등 사용자의 상황에 따라 웹 사용이 제한될 수 있다는 점도 고려해야 한다. 
다양한 환경에서 사용자가 쉽고 빠르게 운동을 기록할 수 있도록 인터페이스를 최적화하고, 제한적인 웹 사용 대안사항도 고려해야할것 같다.

# 6. Glossary
| **Terms (용어)** | **Description (설명)** |
| --- | --- |
| **Ex-Log** | 본 프로젝트의 명칭으로, 운동 기록(Exercise Log)에 게이미피케이션 요소를 결합한 서비스를 의미함. |
| **경험치 (Exp)** | 사용자가 운동을 기록할 때마다 획득하는 포인트로, 티어 상승과 랭킹 결정의 핵심 지표임. |
| **티어 (Tier)** | 누적 경험치에 따라 분류되는 사용자의 등급 체계로, 게임의 랭크 시스템을 벤치마킹함. |
| **칭호 (Title)** | 특정 운동 목표(예: 하체 운동 50회) 달성 시 수여되는 명예 보상 시스템임. |
| **잔디 (Activity Map)** | GitHub의 기여도 그래프처럼, 운동한 날짜를 시각적인 색상 칸으로 표시해주는 활동 이력 시각화 기능임. |
| **이메일 인증** | 회원가입 시 유효한 이메일인지 확인하기 위해 Redis를 활용해 일회용 번호를 검증하는 절차임. |
| **점진적 과부하** | 근력 성장을 위해 운동의 강도나 양을 단계적으로 늘려가는 원리로, 본 서비스의 기록 기능이 지향하는 핵심 가치임. |
| **리텐션 (Retention)** | 사용자가 서비스를 이탈하지 않고 지속적으로 다시 방문하여 이용하는 비율을 의미함. |
| **유효성 검사 (Validation)** | 사용자가 입력한 데이터(횟수, 시간 등)가 상식적인 범위 내에 있는지 서버에서 확인하여 데이터의 신뢰성을 확보하는 과정임. |
| **Redis** | 이메일 인증 코드와 같이 속도가 중요하고 일시적으로 저장되는 데이터를 처리하기 위한 고성능 메모리 데이터베이스임. |
| **데이터 테이블** | 유저 정보, 운동 로그, 칭호 데이터 등이 영구적으로 저장되는 관계형 데이터베이스(MySQL 등)의 저장 구조임. |
